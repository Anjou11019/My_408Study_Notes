## 1.3 操作系统的运行环境

1.下列说法中正确的是（2，4）
	1. 批处理的最大缺点是需要大量内存
	2. 当计算机提供了内核态和用户态时，I/O指令必须在内核态下执行
	3. 操作系统采用多道程序设计技术的最主要原因是提高CPU和外部设备的可靠性
	4. 操作系统中，通道技术是一种硬件技术
 - **解析**：1错误：批处理的主要缺点是**缺少交互性**；2正确：输入/输出指令是特权指令；3错误：多道性是为了**提高系统利用率和吞吐量**而提出的；4：正确：I/O通道实际上是一种**特殊的处理器**，它具有执行I/O指令的能力，并通过执行通道程序来控制I/O操作

2.**中断**是操作系统必须提供的功能

3.在中断发生后，中断的处理程序属于**操作系统程序**
 - **解析**：中断处理程序是操作系统专门为处理中断事件而设计的程序。中断发生时，若被中断的是用户程序，则CPU**由用户态转向内核态**，在内核态下进行中断处理；若被中断的是低级中断，则CPU仍保持在内核态。被中断的可能是用户程序，但中断的处理程序一定是OS程序

4.CPU的状态分为用户态和内核态，从用户态转到内核态的唯一途径是**中断**
 - **解析**：CPU通过程序状态字寄存器中的某个标志位来标志当前的状态，但修改程序状态字的指令本身就属于**特权指令**。当CPU执行到一条访管指令或陷阱指令时，会引起访管中断或陷阱中断，CPU会保存断点和其他上下文环境，然后切换到内核态。也就是说，从用户态转换到内核态，**不是通过指令来修改CPU的状态标志位，而是CPU在中断时自动完成的**

5.下列指令，可以在用户态执行的是（3，4）
	1. 置时钟指令
	2. 停机指令
	3. 存数指令
	4. 寄存器清零指令
 - **解析**：**直接的资源管理指令**（设置时钟、启动/关闭硬件设备、切换进程、设置中断等）、**系统状态修改指令**（修改中断向量表、切换CPU的运行模式等）、**系统控制指令**（停机指令、重启指令），都属于**特权指令**，必须在内核态执行。普通的数据处理指令、流程控制指令、读操作指令都不会影响系统的安全和整体状态，可以在用户态运行

6.下列指令，必须在内核态执行的是（2，3，5）
	1. 陷入指令
	2. 系统调用指令
	3. 开中断指令
	4. 转移指令
	5. 中断屏蔽字设置指令
 - **解析**：
    - 用户程序通过陷入指令让CPU进入内核态，因此在用户态执行；
    - 虽然调用的发生可能在用户态（区分调用和执行），但系统调用指令必然在内核态执行；
    - 开中断/关中断都属于特权指令，能影响到系统的运行，必须在内核态运行；
    - 无论是分支跳转还是无条件转移指令，都属于普通的程序流程控制指令，在用户态执行
    - 中断屏蔽字寄存器属于中断控制器一部分，而中断控制器是属于I/O接口的，因此设置中断屏蔽字寄存器的操作就相对于对I/O接口中I/O端口进行设置，属于I/O指令，必须在内核态执行

7.当CPU处于内核态时，它可以执行的指令是**除访管指令外的全部指令**
 - **解析**：访管指令（trap）是在用户态下执行的一条指令。用户程序在用户态下要请求操作系统内核提供服务，就有意识的使用访管指令引起访管中断，从而将控制权交给内核。在内核态，CPU可以执行除访管指令外的任何指令

8.下列选项中，在用户态执行的是**命令解释程序**
 - 解析：缺页处理程序和时钟中断处理程序都是中断，在内核态执行，进程调度是操作系统内核进程，无须用户干预，在内核态执行；而命令解释程序属于命令接口，是面对用户的，在用户态执行

9.中断处理和子程序调用都需要压栈，以保护现场，中断处理一定会保存而子程序调用不需要保存其内容的是**程序状态字寄存器**
 - **解析**：子程序调用**不改变程序的状态**，因为子程序调用是编译器可控的流程，而中断不是。
   以`if(a==b)`举例：
	 - 该程序一般包含一条测试指令，以及一条根据标志位决定是否需要跳转来调用子程序的指令
	 - 编译器不在两条指令中间插入任何子程序调用代码，因此标志位不变，但中断随时可能发生，导致标志位改变
	 - 具体而言，执行`if(a==b)`时，会进行a-b的操作，并生成相应的标志位，进而根据标志位来判断是否发生跳转
	 - 假设刚好在生成相应的标志位后发生了中断，若不保存PSW的内容，则后续根据标志位来进行跳转的流程可能会发生错误。
	 - 但若进行了子程序调用，则说明已经根据a-b的标志位发生了跳转，此时PSW的内容已经无意义而无需保存了
- 综上所述，中断处理和子程序调用都可能使得PSW发生变化，但中断处理程序执行完返回后，可能会用到PSW原来的内容，子程序执行完返回后，一定不需要用到PSW原来的内容

10.处理外中断时，应该由操作系统保存的是**通用寄存器的内容**
 - **解析**：PC值由中断隐指令自动保存，而通用寄存器内由操作系统保存。快表TLB和Cache中的内容在外中断处理过程中通常无须保存，直接置有效位为0即可

11.假定下列指令已经装入指令寄存器，则执行时不可能导致CPU从用户态到核心态的是`Not R0`;寄存器R0内容取非
 - **解析**：DIV R0,R1;可能会除零导致除零异常；INT n;产生软中断，用于触发中断并跳转到中断处理程序；MOV R0,addr;将addr中的内存数据放入寄存器R0，可能会导致缺页异常

12.定时器产生时钟中断后，由时钟中断服务程序更新的部分内容是（1，2，3）
	1. 内核中时钟变量的值
	2. 当前进程占用CPU的时间
	3. 当前进程在时间片内的剩余执行时间
 - **解析**：时钟中断主要工作是**处理和时间有关的信息及决定是否执行调度程序**，和时间有关的所有信息包括系统时间、进程的时间片、延时、使用CPU时间、各种定时器

13.下列关于系统调用的叙述中，正确的是（1，2，4）
	1. 在执行系统调用服务程序的过程中，CPU处于内核态
	2. 操作系统通过提供系统调用避免用户程序直接访问外设
	3. 不同的操作系统为应用程序提供了统一的系统调用接口
	4. 系统调用是操作系统内核为应用程序提供服务的接口
 - **解析**：设备管理属于操作系统的职能之一，包括对输入/输出设备的分配、初始化、维护等，用户程序需要通过系统调用使用操作系统的设备管理服务

14.下列与中断相关的操作中，由操作系统完成的是（2，3，4）
	1. 保存被中断程序的中断点
	2. 提供中断服务
	3. 初始化中断向量表
	4. 保存中断屏蔽字
- **解析**：
	- 当CPU检测到中断信号后，由硬件自动保存被中断程序的断点\[程序计数器（PC）和程序状态字寄存器（PSW）]，1错误；
	- 之后，硬件找到该中断信号对应的中断向量，中断向量指明中断服务程序入口地址（各个中断向量统一存放在中断向量表中，该表由操作系统初始化），3正确
	- 接下来开始执行中断服务程序，保存中断屏蔽字、保存各个通用寄存器的值，并提供与中断信号对应的中断服务，中断服务程序属于操作系统内核

15.下列指令中，只能在内核执行的指令是**I/O指令**
 - **解析**：常见的特权指令有：**有关对I/O设备操作的指令**；**有关访问程序状态的指令**；**存取特殊寄存器的指令**；**其他指令**

16.执行系统调用涉及下列操作，其中由操作系统完成的是（2，3）
	1. 保存断点和程序状态字
	2. 保存通用寄存器的内容
	3. 执行系统调用服务例程
	4. 将CPU模式改为内核态
- **解析**：发生系统调用时，CPU通过执行软中断指令将CPU的运行状态从用户态切换为内核态，这个过程与中断和异常的响应过程相同，由硬件负责保存断点和程序状态字，并将CPU模式改为内核态。然后执行操作系统内核的系统调用入口，该内核程序负责保存通用寄存器的内容，再调用执行特定的系统调用服务例程
- 综上，1，4由硬件完成；2，3由操作系统完成

## 1.4 操作系统结构、1.5 操作系统引导、1.6 虚拟机

1.下列说法中，**功能分层设计**不适合描述微内核操作系统
 - **解析**：功能分层设计是分层式OS的特点。通常从4个方面描述微内核OS：
	   1. 内核足够小
	   2. 基于客户/服务器模式
	   3. 应用机制与策略分离原理
	   4. 采用面向对象技术

2.计算机操作系统的引导程序位于**硬盘**中
 - **解析**：操作系统的引导程序位于磁盘活动分区的引导扇区中。引导程序分为两种：
    - 一种位于ROM中的自举程序（BIOS的组成部分），用于启动具体的设备
    - 另外一种位于装有操作系统硬盘的活动分区的引导扇区中的引导程序（启动管理器），用于引导操作系统

3.虚拟机的实现离不开虚拟机管理程序（VMM），下列关于VMM说法正确的是（1，3）
	1. 第一类VMM直接运行在硬件上，其效率通常高于第二类VMM
	2. VMM的上层需要支持操作系统的运行、应用程序的运行，因此实现VMM的代码量通常大于实现一个完整操作系统的代码量
	3. VMM可将一台物理机器虚拟化为多台虚拟机器
	4. 为了支持客户操作系统的运行，第二类VMM需要完全运行在最高特权级
 - **解析**：VMM的功能没有操作系统的功能复杂，其代码量少于一个完整的操作系统第一类VMM运行在最高特权级（内核态），而第二类VMM和普通应用程序的地位相同，通常运行在较低特权级（用户态）

## 2.1 进程线程简介

1.下列关于并发进程特性的叙述中，正确的是（B）
	A. 进程是一个动态过程，其生命周期是连续的
	B. 并发进程的运行结果具有不可再现性
  - **解析**：并发进程可能因为等待资源或被抢占CPU而等待，所以其生命周期并不连续。执行速度会影响进程之间的执行顺序和内存冲突问题，从而导致不同的执行结果。并发进程之间存在相互竞争和制约，导致每次运行都产生不同的结果

2.并发进程执行的相对速度是**与进程调度策略有关**
 - **解析**：并发进程执行的相对速度与进程的调度策略有关，因为调度策略决定了哪些进程可以获得处理机，以及获得的时间长短，从而影响进程的执行速度和效率

3.下列关于进程的程序的说法错误的是（**B**）
	A. 一个进程在其生命周期中可执行多个程序
	B. 一个进程在同一时刻可执行多个程序
	C.一个程序的多次运行可形成多个不同的进程
	D.一个程序的一次执行可产生多个进程
- **解析**：一个进程可以顺序执行一个或多个程序，只需在执行过程中改变其CPU状态和内存空间即可，但不能执行多个程序，B错误。一个程序可对应多个进程，即多个进程可以执行同一程序（同一个文本编辑器可以被多用户或多窗口运行），一个程序在执行过程中也可能产生多个进程（可以调用fork(),create()生成子进程，从而实现并发处理和分布式运算）

4.下列选项中导致创建新进程的是（1，2，3，4）
	1. 用户登录
	2. 高级调度发生时
	3. 操作系统响应用户提出的请求
	4. 用户打开了一个浏览器程序
 - **解析**： 用户登录时，操作系统会为用户创建一个登录进程，用于验证用户身份和提供用户界面。高级调度即作业调度，会从后备队列上选择一个作业调入内存，并为之创建相应的进程。操作系统响应用户提出的请求时，通常会为用户创建一个子进程，用于执行用户指定的任务或程序。用户打开一个浏览器是也是一种操作系统响应用户请求的情况，同样会创建一个新进程

5.单处理机系统中，若同时存在10个进程，则处于就绪队列中的进程最多有**9个**
- **解析**：在单处理器系统中，不可能出现10个进程都处于就绪态的情况。但9个处于就绪态，一个处于运行态是有可能的。此外，还有可能10个都在阻塞态（死锁）

6.系统进程所请求的一次I/O操作完成后，将使得进程状态**由阻塞态转为就绪态**
 - **解析**：I/O操作完成之前进程一直在等待结果，处于阻塞态；完成后，等待事件就绪，转为就绪态

7.用户进程被建立后，**随着进程运行的正常或不正常结束而撤销**
 - **解析**：进程有自己的生命周期，不会一直存在与系统中，也不一定需要用户显示的撤销。进程在时间片结束时只是就绪，而不是撤销。阻塞和唤醒是进程生存期的中间状态。进程可在完成时撤销，或出现内存错误等时撤销

8.同一程序经过多次创建，运行在不同的数据集上，形成了**不同的进程**
 - **解析**：进程是程序的一次执行过程，它不仅包括程序的代码，还包括程序的数据和状态。同一个程序经过多次创建，运行在不同数据集上，会形成不同的进程，它们之间没有必然的联系

9.进程自身决定**从运行态到阻塞态**
- **解析**：进程在运行过程中，若主动执行如read()、wait()等系统调用，以请求I/O操作或等待资源，会自行触发从运行态到阻塞态的转换，而其他状态的转换均由内核控制：
   - 转为就绪态通常由**时间片耗尽**或**高优先级进程抢占**引起的，
   - 就绪态转换为运行态由**调度程序决定**，
   - 阻塞态转为就绪态则依赖**外部事件触发**。
   - 这些转换均非自身所能决定的

10.下列关于信号发生和处理的描述中，错误的是（D）
	A.一个进程可以给自己发信号
	B.操作系统的内核可以给进程发送信号
	C.操作系统的内核对每种信号都有默认处理程序
	D.用户可以对每种信号自定义处理函数
 - **解析**：有些信号是不能被用户自定义处理函数的，只能执行操作系统默认的处理程序

11.下列叙述中，正确的是**引入线程可提高程序并发执行的程度，可进一步提高系统效率**
- **解析**：线程是进程内一个相对独立的执行单元，但不能脱离进程单独运行，只能在进程中运行。引入线程是为了减少程序执行的时空开销。一个进程可以包含多个线程

12.下列选项中（D）不是线程的优点
	A.提高系统并发度
	B.节约系统资源
	C.切换开销小
	D.增强进程安全性
- **解析**：线程相比进程的核心优势在于**粒度更细、开销更小**：一个进程可以包含多个线程并发执行，显著提升系统并发性；线程不拥有独立的内存空间，仅需少量的私有数据结构，资源占用远低于进程；线程切换无需切换地址空间，仅保存和恢复少量上下文，开销更低；但因为线程共享进程资源，一个线程出错可能影响整个进程，无法增强进程安全性

13.在下列描述中，**键盘驱动程序为每个正在运行的应用配备一个线程，用以响应应用的键盘输入**并不是多线程系统的特长
 - **解析**：整个系统只有一个键盘，而且键盘输入是人的操作，速度比较慢，完全可以使用一个线程来处理整个系统的键盘输入

14.一个进程**从运行态变为就绪态**必然会引起进程切换
 - **解析**：进程切换是指CPU调度不同的进程执行，当一个进程从运行态转变为就绪态时，CPU调度另一个进程执行引起进程切换

15.下面关于用户级线程和内核级线程的描述中，错误的是（B）
	A.采用轮转调度算法，进程中设置内核级线程和用户级线程的效果完全不同
	B.跨进程的用户级线程调度也不需要内核参与，控制简单
	C.用户级线程可以在任何操作系统中运行
	D.若系统中只有用户级线程，则CPU的调度对象是进程
- **解析**：用户级线程的调度仍然以进程为单位，各个进程轮流执行一个时间片。假设进程A包含1个用户级线程，而进程B包含100个用户级线程，此时进程A中单个线程的运行时间将是进程B中各个线程平均运行时间的100倍；内核级线程的调度是以线程为单位的，各个线程轮流执行一个时间片，假设同样A包含1个内核级线程，而B包含100个，那么B的运行时间是A的100倍，A正确。用户级线程的调度单位是进程，跨进程的线程调度仍然需要内核支持，B错误。用户级线程是由函数库或用户程序实现的，不依赖于操作系统的支持，C正确。用户级线程对操作系统是透明的，CPU的调度对象仍然是进程

16.下列说法中正确的是**不管系统中是否有线程，进程都是拥有资源的独立单位**
 - **解析**：引入线程后，进程仍然是资源分配的单位。内核级线程是处理器调度和分派的单位，线程本身不具备资源，它可以共享所属进程的全部资源

17.某程序的虚拟地址空间包括**内核区、用户栈、运行时堆、可读写数据段和只读代码段**
有以下C语言代码
```cpp
char *ptr;
void main(){
	int length;
	ptr = (char *)malloc(100);
	scanf("%s",ptr);
	length = strlen(ptr);
	printf("length = %d\n",length);
	free(ptr);
}
```
main()函数的代码位于哪个区域？其直接调用的哪些函数的功能需要通过执行驱动程序实现？
 - **解析**：main()函数的代码存储在**只读代码段中**。scanf()和printf()虽然为标准库函数，但其底层通过系统调用访问设备，最终依赖**设备驱动程序**完成实际I/O操作
